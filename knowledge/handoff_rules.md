# 部门交接规则

## 总原则

这家机器人公司必须避免“每个岗位都很努力，但没人知道下一步该交给谁”。

所以每个部门都必须满足：

- 有明确输入
- 有明确输出
- 有明确交接条件

## 交接链路总览

`CEO -> 市场策略负责人 -> 合规审校官 -> 各平台编辑 -> 运营分析师 -> 线索经理 -> 试用成功经理 -> 成交经理 -> CEO`

## 1. CEO -> 市场策略负责人

输入：

- 本周主题
- 本周目标
- 本周重点渠道

输出：

- 周策略草案
- 选题清单
- CTA 方向

交接条件：

- 本周主题明确
- 渠道优先级明确
- 至少有 3 个核心内容支柱

## 2. 市场策略负责人 -> 合规审校官

输入：

- 周策略草案
- 内容角度
- 产品卖点

输出：

- 合规可发布底稿

交接条件：

- 不出现荐股
- 不出现收益承诺
- 不出现资质越界

## 3. 合规审校官 -> 各平台编辑

输入：

- 合规底稿

输出：

- X 内容
- 小红书内容
- 公众号内容
- 雪球内容

交接条件：

- 每个平台都有对应版本
- 每个平台 CTA 明确
- 每个平台口吻符合平台特性

## 4. 各平台编辑 -> 运营分析师

输入：

- 各平台内容稿

输出：

- 一周发布清单
- 渠道日历
- 指标建议

交接条件：

- 每个平台内容完成
- 发布时间建议明确
- 本周目标动作明确

## 5. 运营分析师 -> 线索经理

输入：

- 发布计划
- CTA
- 评论关键词

输出：

- 线索池
- 线索优先级
- 线索下一步动作

交接条件：

- 线索已分层
- 每条线索有 next action
- 标注来源渠道

## 6. 线索经理 -> 试用成功经理

输入：

- `trial` 阶段用户
- 申请试用用户

输出：

- 首次跟进动作
- 试用激活提醒
- 用户问题记录

交接条件：

- 用户已申请试用
- 已给出首条试用引导

## 7. 试用成功经理 -> 成交经理

输入：

- 已激活试用用户
- 明确表达兴趣的试用用户

输出：

- 高意向用户名单
- 成交建议
- 推荐成交话术

交接条件：

满足以下任意 2 条即可交接：

- 已使用 2 个以上核心功能
- 主动问价格
- 主动问正式版区别
- 明确表达“这个对我有帮助”
- 连续有互动

## 8. 成交经理 -> CEO

输入：

- 本周成交推进结果

输出：

- 高意向名单
- 付费结果
- 卡点总结

交接条件：

- 有清晰成交结论
- 有失败原因复盘
- 有下周优化建议

## 线索定义规则

### cold

- 仅浏览
- 仅关注
- 无明显互动

### warm

- 评论
- 私信
- 点击试用
- 问功能

### trial

- 已申请试用

### active_trial

- 已完成首次有效使用

### hot

- 明确问价格
- 明确问正式版
- 明确表达购买意愿

### paid

- 已付费

## 升级规则

### warm -> trial

满足任一：

- 点击试用链接
- 回复“试用”
- 填表申请

### trial -> active_trial

满足任一：

- 使用教练指令
- 使用执行计划
- 使用持仓诊断

建议最好满足 2 项以上

### active_trial -> hot

满足任意 2 条：

- 主动问价格
- 问正式版区别
- 说“有帮助”
- 连续两次互动

### hot -> paid

- 已成交

## 交接失败怎么处理

### 情况 1：内容做完但没有 CTA

处理：

- 退回编辑重写

### 情况 2：线索池里没有 next action

处理：

- 退回线索经理补动作

### 情况 3：试用用户很多但没人跟

处理：

- CEO 直接把试用激活列为第一优先级

### 情况 4：高意向用户没人推进成交

处理：

- 成交经理必须每日清空 hot 列表

## 最终规则

每一次交接都必须回答清楚：

1. 现在这个人处于哪一层
2. 下一步最该做什么
3. 该交给谁
